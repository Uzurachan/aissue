# Issue詳細表示

読み取り専用の表示なので、ヘルパースクリプトは使わずこのSkillの指示に従って直接 `index.md` を読んで表示する(ファイルを変更しないため、`new_issue.py`等のような機械的処理への切り出しは不要)。

## 対象Issueの特定

- ユーザーがIssue ID(例: `0001`)を明示していればそれを使う。
- 「一番上の」「次にやるやつ」のように順序で指定された場合は、`python3 list_issues.py` の結果(`order`昇順)から該当するIssueを特定する。
- タイトルやキーワードでの指定の場合は `issues/*/index.md` のtitleやtagsから探す。
- 複数候補があり一意に絞れない場合は、ユーザーに確認する。

## 出力フォーマット

以下の順で表示する。

1. **見出し**: `# <id>: <title>`
2. **メタデータテーブル**(frontmatterの値をそのまま並べる):

   ```
   | 項目 | 値 |
   |---|---|
   | status | ... |
   | priority | ... |
   | tags | ... |
   | assignee | ... |
   | order | ...(未設定なら `-`) |
   | created | ... |
   | updated | ... |
   ```

3. **本文**: frontmatterより下の本文を、要約・加工せずそのまま表示する。
4. **添付ファイル一覧**: `<issue_dir>/<attachments_dir>/` を実際にスキャンして、そこに存在するファイルをリスト形式で示す。

   ```
   - `<ファイル名>`(<簡潔な説明>)
   ```

   説明は本文中に同じファイル名への言及があればそこから拾い、なければ説明なしでファイル名のみ表示する。
   **添付ファイルの中身は表示しない**(存在の一覧のみ)。

## 注意点

- 表示専用の操作であり、`index.md`やattachments/の内容を書き換えない。
- 本文を勝手に要約・改変しない(見出し構成や文章はそのまま)。
